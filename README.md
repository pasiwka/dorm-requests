# CampusNet — Система управления заявками общежития

## 1. Цель проекта

Создать современное веб-приложение для автоматизации подачи и обработки заявок в студенческом общежитии с упором на качественный фронтенд: адаптивную вёрстку по БЭМ, SCSS, чистый JavaScript и продуманный UI/UX.

Проект предназначен для демонстрации:

- адаптивной вёрстки по методологии БЭМ
- работы с SCSS (препроцессоры, миксины, переменные)
- создания сложных интерфейсов с анимациями и градиентным фоном
- написания клиентской логики на чистом JavaScript (ES6+)
- взаимодействия с REST API (fetch, async/await)
- проектирования UI/UX в Figma
- разработки бекенда на Node.js + Express
- проектирования базы данных в SQLite (3 нормальная форма)

---

## 2. Функциональные требования

### 2.1. Структура базы данных

База данных спроектирована в SQLite и приведена к третьей нормальной форме.

**Таблица users**

| Поле | Тип | Описание |
|------|-----|----------|
| id | INTEGER (PK) | Уникальный идентификатор |
| phone | TEXT (UNIQUE) | Номер телефона (логин) |
| password | TEXT | Пароль |
| role | TEXT | student, admin |
| first_name | TEXT | Имя пользователя |
| last_name | TEXT | Фамилия пользователя |
| created_at | DATETIME | Дата регистрации |

**Таблица requests**

| Поле | Тип | Описание |
|------|-----|----------|
| id | INTEGER (PK) | Уникальный идентификатор заявки |
| user_id | INTEGER (FK) | ID студента, создавшего заявку |
| category_id | INTEGER (FK) | Категория заявки |
| description | TEXT | Описание проблемы |
| status | TEXT | pending, approved, rejected, done |
| room_id | INTEGER (FK) | ID комнаты проживания |
| created_at | DATETIME | Дата создания заявки |
| updated_at | DATETIME | Дата последнего обновления |
| admin_comment | TEXT | Комментарий администратора |


## 3. API эндпоинты
| Метод | URL | Доступ | Описание |
|-------|-----|--------|----------|
| GET | /api/requests | ADMIN | Получить все заявки |
| GET | /api/requests/user/{userId} | STUDENT | Заявки текущего студента |
| POST | /api/requests | STUDENT | Создать заявку |
| PUT | /api/requests/{id}/status | ADMIN | Изменить статус |

## 5. Фронтенд и БЭМ-методология

### 5.1. Синтаксис БЭМ

| Синтаксис | Пример | Значение |
|-----------|--------|----------|
| block | student, filter | Независимый компонент |
| block__element | student__card, filter__item | Часть блока |
| block--modifier | button--primary, step--active | Состояние или модификатор |

### 5.2. Пример БЭМ-разметки

```html
<div class="student">
    <div class="student__card">
        <h1 class="student__title">Мои заявки</h1>
        <ul class="filter">
            <li class="filter__item filter__item--active">Все</li>
            <li class="filter__item">В ожидании</li>
            <li class="filter__item">Принято</li>
        </ul>
        <button class="button button--primary">Создать заявку</button>
    </div>
</div>
```
### 5.4. Статусы заявок

| Статус | Цвет | CSS-класс |
|--------|------|-----------|
| В ожидании | Жёлтый | status-pending |
| Принята | Зелёный | status-approved |
| Отклонена | Красный | status-rejected |
| Выполнена | Синий | status-done |


## 6. Функциональные возможности

**Для студента:**

- Вход в систему
- Просмотр своих заявок с фильтрацией по статусу
- Создание новой заявки (выбор категории: Сантехник, Электрик, Мебель)
- Отслеживание статуса заявки в реальном времени

**Для администратора:**

- Просмотр всех заявок в таблице
- Изменение статуса заявки через выпадающий список
- Поиск по имени, категории, описанию, комнате, статусу
- Фильтрация по статусу

**Общие возможности:**

- Адаптивная вёрстка (мобильные устройства, планшеты, десктопы)
- Анимированный градиентный фон
- Плавные переходы и hover-эффекты
- Кастомный скроллбар

## 7. Запуск проекта

```bash
# Установка зависимостей бекенда
cd backend
npm install

# Запуск сервера
node server.js
```
Сервер запустится на http://localhost:3000

Откройте в браузере файл: frontend/pages/login.html
